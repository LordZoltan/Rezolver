    Remarks

    Scopes in Rezolver operate much the same as they do in any IOC framework, but use of them is always entirely optional.

    When a scope is active for a given Resolve(IResolveContext) operation, most objects which are returned from those operations will implicitly be scoped to whichever scope is active when the objects are resolved. Implicitly scoped objects are only disposed of when their containing scope is disposed, and you can have an unlimited number of instances of implicitly scoped objects per scope.

    This target is used to scope the object produced by a target explicitly to a scope, and to ensure that only one instance of that object is produced per scope. Such objects are also not inherited between parent scopes and child scopes.
