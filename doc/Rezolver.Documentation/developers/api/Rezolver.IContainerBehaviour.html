    Interface IContainerBehaviour

    Represents a behaviour for an IContainer (specifically one which also uses an ITargetContainer as the source of its registrations).

    Namespace:Rezolver
    Assembly:Rezolver.dll
    Syntax
    public interface IContainerBehaviour
    Remarks

    While similar to ITargetContainerBehaviour, this is specifically used for new containers.

    A behaviour can be passed to any of the provided ContainerBase-derived types on construction (see the ContainerBase(IContainerBehaviour, ITargetContainer) constructor). If one is not passed then a default is used from the GlobalBehaviours class.

    The actual one used depends whether it's an OverridingContainer or not: Rezolver has global behaviours for the standalone container types (via the ContainerBehaviour property) and for overriding containers (via the OverridingContainerBehaviour property).

    Behaviours extend ITargetContainer-based containers by registering well-known service types in the target container passed to Attach(IContainer, ITargetContainer) - which is the target container from which the container itself will retrieve registrations when resolving instances. The built in IContainer classes resolve these service types (e.g. ITargetCompiler) as part of normal operation, so - by providing alternative or additional registrations - behaviours can extend the underlying container.
