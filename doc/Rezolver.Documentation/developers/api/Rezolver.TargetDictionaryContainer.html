    Remarks

    This container implementation actually stores containers against the types that targets are registered against, rather than simply storing a dictionary of targets. This method allows you to add your own containers against type (instead of the default, which is TargetListContainer) so you can plug in some advanced behaviour into this container.

    For example, decorators are not actually ITarget implementations but specialised ITargetContainer instances into which the 'standard' targets are registered.
