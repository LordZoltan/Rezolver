    The simplest way to create an IContainer, registering targets in it and resolving objects from it couldn't be easier.

    Note

    In Rezolver, registering and resolving services are operations which are handled by two separate interfaces (ITargetContainer and IContainer). The two most-used standard container classes (Container and ScopedContainer) implement both of these interfaces, however, in order to provide the most convenient developer experience.

    So, in order for your application to be able to both register targets in, and resolve objects from, a container, you will need a reference to one of those two container classes.

    Note

    A target (ITarget) is an object stored by an ITargetContainer which contains information about how an object is to be created or retrieved when a container's Resolve operation is called.

    ![NOTE] The core implementation of IContainer is ContainerBase, which is shared by both Container and ScopedContainer - and it is this class which provides the implementations of both IContainer and ITargetContainer.

    For the majority of these code samples, we'll be using the many extension methods for both registration and resolving objects which are provided by numerous extension classes:

    • Registration:
      • AliasTargetContainerExtensions
      • DecoratorTargetContainerExtensions
      • DelegateTargetContainerExtensions
      • ExpressionTargetContainerExtensions
      • MultipleTargetContainerExtensions
      • ObjectTargetContainerExtensions
      • RegisterTypeTargetContainerExtensions
      • ScopedTargetContainerExtensions
      • @Reaolver.SingletonTargetContainerExtensions
    • Resolving:
      • ContainerResolveExtensions

    Simple registration

    The simplest thing we can do is to register a type for which we want an instance to be created when request that type.

    Container container = new Container();
container.RegisterType<MyService>();
var result = container.Resolve<MyService>();
Assert.NotNull(result);
