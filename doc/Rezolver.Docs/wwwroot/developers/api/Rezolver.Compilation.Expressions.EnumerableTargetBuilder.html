    Remarks

    The compiler is capable of producing both lazy-loaded and eager-loaded enumerables, which can be controlled via target container options.

    Lazy vs Eager loading

    The option LazyEnumerables is read from the context for the ElementType of the target. If it is equivalent to true (the Default), then a lazily-loaded enumerable is constructed which will create new instances of each object in the enumerable each time it is enumerated.

    If the option is instead equivalent to false, then all instances will be created in advance, and an already-materialised enumerable is constructed.
